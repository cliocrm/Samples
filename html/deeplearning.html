Simple Deep Learning for Images and Videos

While tackling the problem of creating a minimal data product detail page discussed in this article, I had to look for deep learning algorithms with which products in images and videos are automatically classified, cropped, enhanced, and tagged with keywords/brand names that can be used to generate a product or service description. There are several open-source and paid cloud API solutions that do this. Examples of paid cloud services are Google Vision API and AutoML, Microsoft Computer Vision in Azure, Amazon Rekognition and SageMaker APIs, IBM Watson Image recognition, Imagga, etc. Unfortunately, none of them was perfect for serving all of my requirements. I started looking for open source solutions for images to identify the primary object from the user-provided images, crop it, or compress it perfectly if needed, and improve the overall image quality. After finding a few of them and comparing them in terms of precision, the efficiency of the deep learning algorithm, and cost per API call, I came to the conclusion that it is indeed good to divide each function amongst the best in each category. For instance, Google Vision API was the best in providing the relevant keywords for my set of test images. For cropping, I used an open-source solution in GitHub that also enhances the color of the image and the speed was the best. The link to the repo is https://github.com/jwagner/smartcrop.js/
Image resizing is another problem since it was often seen from my experience with Snapdeal's product images provided by the merchants that they are often of varied sizes. Resizing it to fit a standard-sized image frame can be as easy as simply zooming in or out, or cropping it, or compressing it. In each of these, it is important to keep in mind that the primary object for sale, for instance, should not be distorted. To solve this the content-aware image processing algorithms were developed, the best of which is the seam carving algorithm. Same for image auto-cropping.
Collecting keywords for the object is not a very difficult task, the main pain point is to use it to generate text for product description and marketing content. One source of knowledge to get this done is mentioned in the article https://www.practicalecommerce.com/Generate-Product-Descriptions-with-Natural-Language-Software and the approach there has nothing to do with deep learning. It merely collects parameters and attribute data from the merchant and uses that with an inbuilt template to generate the text. The same logic is applied by https://ginnie.ai/ https://adzis.com/ and https://itemscribe.com/ that charges some amount per description generated. We need a strong machine learning and NLG model to generate human-readable grammatically correct text with desired keywords and in a given context like marketing a product or service to a target audience. This is still a problem to be solved by us. But there are several research papers that have attempted to build frameworks for this, like
A Statistical Framework for Product Description Generation
Towards Knowledge-Based Personalized Product Description Generation in E-commerce
A product description plays an important role in getting customers' attention and in product or service import functionality for our application it is particularly important since we need to present it as unique as possible. Using the customer reviews for creating the best possible headline can be perfected as discussed in the article: https://dl.acm.org/doi/10.1145/3308558.3313532
Combined with that is to use the free projects as much as possible and call the individual APIs for each function if we are not developing the ML/DL solutions ourselves. Each task can be broken down into microservices - image/video processing, text and item detail generation, and so on.
Similar functions for e-commerce videos, identifying the primary object from periodic snapshots, motion, and actions if any, processing it to remove unwanted clips, beautifying it, and adding background audio clips like music or even the product description using text to speech APIs. Adding extra information like related items at the end or store name and brand video. Generating videos from static product images is already being done by many merchants though it is not appealing to all kinds of audiences.
