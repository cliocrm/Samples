Customizing Jitsi Meet for Video Conferencing

The open-source projects that are matured and relevant in the current business needs can be customized to include the company logo and relevant text.
These articles from Jitsi and others like https://www.liquidweb.com/kb/how-to-install-jitsi-meet-on-ubuntu-18/ are easy to follow. But from my experience in GCP, you can have an instance that can host the application for three months using these steps:
(Please note that these steps were relevant at the time of writing this article. For any changes in the steps or in the references mentioned, please verify before proceeding.)
1. Sign up in GCP using your credit card and get the free credits.
2. Buy a basic .xyz domain from GoDaddy or any other domain provider.
3. Create a project in your GCP account and add a Compute Engine instance with 15GB RAM and 4 vCPUs. This is to ensure optimum performance with increasing load. Make other necessary config changes like allowing http and https traffic, and Ubuntu image preferably, enabling the firewall and allowing traffic from ports 4443, 10000, 443, 80, OpenSSH, Apache, and others. You will not have to run the commands later if you do this in the beginning itself or else you will have to use these commands:
sudo ufw enable
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 4443
sudo ufw allow 10000
sudo ufw allow OpenSSH
sudo ufw allow "Apache Full" ---> this is after installing apache2 in the machine and checking http://:80 for the Apache default home page. Alternatively, Nginx or any other web server supporting LetsEncrypt can be used.
4. SSH into the machine or open the browser console and type
sudo hostnamectl set-hostname
e.g. sudo hostnamectl set-hostname vc.cliosuite.xyz

5. sudo vi /etc/hosts
Add
127.0.0.1
localhost
Save and quit

6. wget https://download.jitsi.org/jitsi-key.gpg.key (you will have to install wget in a Debian instance)

7. sudo apt-key add jitsi-key.gpg.key

8. sudo vi /etc/apt/sources.list.d/jitsi-stable.list
Add deb https://download.jitsi.org stable/

9. sudo apt update
sudo apt install jitsi-meet

10. Enter your name on prompt. And choose to Generate letsencrypt ssl as a self-signed certificate.

11. Once this is done, enter
sudo add-apt-repository ppa:certbot/certbot

12. sudo apt install certbot

13. sudo /usr/share/jitsi-meet/scripts/install-letsencrypt-cert.sh and enter an email on prompt.
On successful installation of the SSL you can check by using the URL https:// and the out-of-the-box page of Jitsi will be loaded.
If for any reason, the SSL certificate domain verification step fails, you will have to follow the steps for applying the LetsEncrypt SSL to a domain for Apache in Ubuntu or whatever is your env. The blog "Getting Started with Web Application development for the cloud" discusses in detail the current challenges in using this free SSL provider.
Please note that there is actually a requirement for having a script in every machine for the auto-renewal of this certificate. The validity is for 3 months only.

Now to customize the instance, you need to change a couple of files.
1. For background image of the welcome page, go to /usr/share/jitsi-meet/css and change the
.welcome {
background-image: url(/images/bg3.gif); ... } to include your image in /images folder.
2. For all custom text, I have changed /usr/share/jitsi-meet/lang/main-enGB.json and /usr/share/jitsi-meet/lang/main.json. The changes will automatically load.
Chrome had issues before which are now fixed after contacting their GCP and Chrome support groups on Twitter. Safari still has issues and does not reflect the text changes. Mobile Android Chrome also does the same.
Further customizations include setting a password programmatically, allowing only authorized users to access the application, and integrating the Etherpad text editor.
